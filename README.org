social-scheduling platform.

* Quick Start
1. =npm install=
2. *Environment variables:* copy =.env.example= → =.env= at the repo root and fill in
   each value (the template is annotated per-variable). Never commit =.env= (gitignored;
   see CLAUDE.md). Key groups: Supabase (=SUPABASE_URL=, =SUPABASE_SERVICE_KEY=,
   =VITE_SUPABASE_ANON_KEY=, =SUPABASE_DB_URL=), AI (=GEMINI_API_KEY=), Email
   (=RESEND_API_KEY=), Cron (=CRON_SECRET= — must match the Supabase Vault =cron_secret=;
   see =supabase/RUNBOOK.org=), Social OAuth (LinkedIn / Twitter / Facebook + Instagram /
   Google Calendar), Security (=CREDENTIAL_ENCRYPTION_KEY=, =OAUTH_STATE_SECRET=). In
   production, set these in Vercel → Settings → Environment Variables, then redeploy;
   OAuth redirect URIs must use the live domain.
3. Frontend dev server: =npm run dev= (Vite).
4. Backend: =node server/index.js= (Express API).
5. Background worker: =node server/worker.js= (scheduled posts / metrics).
